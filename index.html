El Corazón Empírico

1. Transparencia

Sin métricas ocultas. Artefactos visibles para todos.

2. Inspección

Auto-auditoría constante del progreso y el código.

3. Adaptación

Corrección de rumbo inmediata si algo falla.

Cuándo NO usar Scrum

Simple (Lineal)

Ej: Instalar servidor.
➡ Mejor usar Cascada.

Complejo (Incierto)

Ej: Software nuevo, Marketing.
➡ El dominio perfecto para Scrum.

Caótico (Emergencias)

Ej: Servidor caído (HelpDesk).
➡ Mejor usar Kanban puro.

Patologías

El "Sprint Zero" y no usar XP (TDD). Generan deuda técnica y Cascada disfrazada.

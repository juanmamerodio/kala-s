Inspección Diaria

Daily Scrum

15min

No es un reporte de estado para el jefe. Es una reunión exclusiva para los Developers para sincronizar el trabajo y ajustar el plan hacia el Sprint Goal.

Developer 1

Terminé la API de pagos ayer. Hoy arranco con los tests de integración.

Developer 2

Yo integro el frontend con tu API. Vamos perfecto para el Goal.

Developer 3

Tengo un impedimento: infraestructura no me da acceso a la DB de QA. ¿Me destrabás, Scrum Master?

Cierre del Ciclo

Review & Retrospective

Sprint Review

Se inspecciona el Incremento. No es una presentación de PowerPoint aburrida, es probar el producto real con los stakeholders y adaptar el Backlog juntos.

Sprint Retrospective

Se inspecciona al Equipo. Procesos, herramientas, relaciones. Se busca de forma honesta al menos una mejora accionable para el próximo Sprint.

Análisis Crítico

Cuándo NO usar Scrum

Basado en el marco Cynefin, Scrum brilla en entornos complejos, pero falla estrepitosamente en otros escenarios operativos.

  • Problemas SimplesMigración lineal de datos o armado de PCs. Cascada es mejor y más barato.
  • Emergencias (Caótico)Soporte técnico o HelpDesk. El trabajo cae en tiempo real, Kanban puro es ideal.
  • Contratos FijosPrecio y alcance cerrados en piedra matan instantáneamente la adaptación ágil.
